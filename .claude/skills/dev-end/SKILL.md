# 개발 종료 (Dev End)

## 오늘의 커밋

```!
git log --since=midnight --oneline
```

## 변경 통계 (미커밋 포함)

```!
git diff --stat HEAD
echo "---"
git status --short
```

## 오늘 날짜

!`date +%F`

## 수행할 작업

1. `docs/daily-logs/{오늘날짜}.md`를 읽는다. (개발 시작 때 만든 "🎯 오늘 계획"이 있으면 참고)
2. 위 커밋 목록과 변경 통계를 근거로 **실제로 한 일**을 사실 기반으로 정리한다. 추측하지 말고 커밋·변경에 드러난 것만 적는다.
3. "🎯 오늘 계획"의 각 항목 완료 여부를 체크박스(`- [x]`)로 갱신한다. 못 끝낸 항목은 "🚧 막힌 점 / 진행 중" 또는 "➡️ 내일 할 일"로 옮긴다.
4. 아래 섹션을 채워 파일을 갱신한다:
   - **✅ 오늘 한 일**: 기능/수정 단위로 간결하게
   - **🚧 막힌 점 / 진행 중**: 미완료, 이슈, 디버깅 중인 것
   - **📝 메모 / 배운 점**: 결정사항, 트레이드오프, 참고 링크 등 (없으면 생략)
   - **➡️ 내일 할 일**: 다음 작업일에 dev-start가 읽을 항목
   - **🔖 커밋 요약**: 오늘 커밋을 `해시 메시지` 한 줄씩
5. 한국어로 간결하게 작성한다. 다 쓰면 어떤 파일을 갱신했는지 사용자에게 알려준다.
